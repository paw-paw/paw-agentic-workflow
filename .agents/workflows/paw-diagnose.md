# PAW Diagnose Workflow

Use this thin workflow only to route a request:

1. Read `docs/README.md` and `AGENTS.md`.
2. Identify whether the request belongs to active `sdd-*` workflow or candidate
   PAW adapter evidence.
3. Check for an existing `sdd/parches/<change-id>/` workspace.
4. Route to the next operation or stop for a decision gate.

This workflow does not replace skills, backlogs, or closure validation.
