# Implementation Presets

Implementation presets are complete, evidence-backed golden paths. They are not
automatic recommendations, adoption records, runtime adapters, or a matrix of
interchangeable technologies.

Each preset fixes one coherent envelope and declares applicability,
contraindications, bounded choices, parameters, subordinate named variants, source
evidence, freshness, and six completeness dimensions. A variant replaces its stated
envelope as a unit; variants cannot be freely combined or decomposed into menu
choices.

Version claims use explicit upstream-supported stable lines. `latest`, implicit
current versions, prereleases, and EOL lines are invalid. Review is due at the earlier
of an upstream lifecycle milestone and 180 days after the recorded review.

The canonical data is `catalog.json`. This guide explains its boundaries and does not
override the manifest.
