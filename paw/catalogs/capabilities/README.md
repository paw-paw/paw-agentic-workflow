# Documentation Capabilities

Capabilities are reusable semantic obligations. A documentation requirement is a
capability applied with scope, applicability, ownership expectations, and evidence.

The canonical IDs, merge strategies, roles, and evidence expectations live in
`catalog.json`. Roles describe document purpose only; they do not assign authority or
precedence.

## Universal Capabilities

Every documentation preset includes the eight capabilities from intent and scope
through authority and conformance.

## Transversal Capabilities

Transversal capabilities enter a family baseline only where the preset declares
them. Component profiles and concerns may add or strengthen them in later catalog
phases.

Merge strategies are capability-specific. There is no universal "strongest value"
operator, and `manual-conflict` always requires a governed human decision.
